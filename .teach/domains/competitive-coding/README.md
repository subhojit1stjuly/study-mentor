# Competitive Coding Domain

This domain provides specialized templates and guidance for mastering competitive programming and algorithmic problem-solving.

## What is Competitive Coding?

Competitive coding involves solving algorithmic problems under time constraints. It's essential for:
- Technical interview preparation (FAANG, etc.)
- Improving problem-solving skills
- Competitive programming contests (Codeforces, AtCoder, etc.)
- Building strong CS fundamentals

## Key Skills

1. **Algorithm Patterns**: Two pointers, sliding window, DP, graphs, etc.
2. **Data Structures**: Arrays, trees, heaps, tries, etc.
3. **Time/Space Complexity**: Big O analysis
4. **Problem-Solving Strategies**: Greedy, divide-and-conquer, backtracking
5. **Implementation**: Clean, bug-free code under time pressure

## Learning Progression

### Phase 1: Foundation (Beginner)
- Basic data structures (arrays, strings, hashmaps)
- Simple patterns (two pointers, sliding window)
- Easy problems on LeetCode/HackerRank
- Time complexity fundamentals

### Phase 2: Development (Intermediate)
- Advanced data structures (trees, graphs, heaps)
- Classic algorithms (BFS, DFS, Dijkstra, DP)
- Medium problems with multiple approaches
- Pattern recognition and optimization

### Phase 3: Mastery (Advanced)
- Hard problems and edge cases
- Contest participation
- Mock interviews
- System design basics (for senior roles)

## Recommended Platforms

1. **LeetCode** - Best for interview prep
   - LeetCode 75, Blind 75, NeetCode 150
   - Company-specific problem sets
   - Mock interviews

2. **Codeforces** - Best for competitive programming
   - Rated contests
   - Strong problem quality
   - Active community

3. **HackerRank** - Good for beginners
   - Structured tracks
   - Interview preparation kits

4. **CodeWars / AtCoder** - Alternative platforms

## Problem-Solving Strategy

1. **Understand**: Read problem 2-3 times, identify constraints
2. **Examples**: Work through examples, identify patterns
3. **Approach**: Think of brute force, then optimize
4. **Complexity**: Analyze time/space before coding
5. **Implement**: Write clean, tested code
6. **Test**: Edge cases, large inputs
7. **Reflect**: What pattern did this use? Related problems?

## Common Patterns (Blind 75 inspired)

1. **Arrays & Hashing**: Two Sum, Group Anagrams
2. **Two Pointers**: Valid Palindrome, Container With Most Water
3. **Sliding Window**: Longest Substring, Max Profit
4. **Stack**: Valid Parentheses, Min Stack
5. **Binary Search**: Search Rotated Array
6. **Linked List**: Reverse, Merge, Cycle Detection
7. **Trees**: DFS, BFS, Validate BST
8. **Tries**: Implement Trie, Word Search
9. **Heap/Priority Queue**: Top K Elements, Merge K Lists
10. **Backtracking**: Combinations, Subsets
11. **Graphs**: Clone Graph, Course Schedule
12. **Dynamic Programming**: Climbing Stairs, Coin Change, LCS
13. **Greedy**: Jump Game, Gas Station
14. **Intervals**: Merge Intervals, Meeting Rooms
15. **Math & Bit**: Power, Reverse Integer

## Assessment Focus

For competitive coding assessments:
- **Conceptual**: Explain algorithms (BFS vs DFS)
- **Applied**: Solve problems with explanations
- **Pattern Recognition**: Identify which pattern to use
- **Teaching**: Explain your solution approach

## Time Commitment

- **Beginner to Interview-Ready**: 3-6 months with 2 hours/day
- **Interview-Ready to Advanced**: 6-12 months with continued practice
- **Maintenance**: 3-5 problems/week to stay sharp

## Success Metrics

- ✅ Can solve Easy problems in <15 minutes
- ✅ Can solve Medium problems in <30 minutes
- ✅ Recognize patterns quickly
- ✅ Pass mock interviews
- ✅ Explain solutions clearly
