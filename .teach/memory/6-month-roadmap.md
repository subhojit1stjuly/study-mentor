## 📍 Month 1-2: DSA Foundation (248 hours)

**Status**: **START HERE** 🚀  
**Primary Skill**: `dsa-mastery` (Phases 0-2)  
**Target**: Build rock-solid foundation in data structures and essential patterns

### Week-by-Week Breakdown

#### **Week 1-2**: Phase 0 & Phase 1.1-1.2 (62 hours)
- **Phase 0**: Complexity analysis, recursion, math for DSA
- **Phase 1.1**: Arrays & strings (sliding window, two pointers, prefix sums)
- **Phase 1.2**: Linked lists (fast/slow pointers, cycle detection, LRU cache)
- **Problems**: 20 Easy + 15 Medium
- **Checkpoint**: Implement dynamic array, solve LRU Cache from scratch

#### **Week 3-4**: Phase 1.3-1.4 (62 hours)
- **Phase 1.3**: Stacks (monotonic stack, valid parentheses, min stack)
- **Phase 1.4**: Queues & deques (circular queue, sliding window maximum)
- **Problems**: 15 Easy + 20 Medium
- **Checkpoint**: Explain when to use stack vs queue vs deque

#### **Week 5-6**: Phase 2.1-2.3 (62 hours)
- **Phase 2.1**: Hash tables (implementation from scratch, collision resolution)
- **Phase 2.2**: Hash applications (two sum patterns, anagram grouping)
- **Phase 2.3**: Comparison sorts (merge sort, quick sort, heap sort)
- **Problems**: 10 Easy + 25 Medium + 5 Hard
- **Checkpoint**: Implement hash map and merge/quick sort from scratch

#### **Week 7-8**: Phase 2.4-2.5 & Review (62 hours)
- **Phase 2.4-2.5**: Non-comparison sorts, sorting applications
- **Review**: Revisit weak areas, reattempt failed problems
- **Problems**: 5 Easy + 20 Medium + 10 Hard
- **Checkpoint**: Comprehensive review quiz, 80%+ success rate on Easy

**Month 1-2 Targets**:
- ✅ 100+ problems solved (50 Easy, 40 Medium, 10 Hard)
- ✅ Master core patterns: two pointers, sliding window, stack/queue tricks
- ✅ Fluent in Big-O analysis
- ✅ Can implement all Phase 0-2 data structures from scratch

## 📍 Month 3: Advanced DSA (124 hours)

**Primary Skill**: `dsa-mastery` (Phases 3-5)  
**Target**: Master trees, graphs, and dynamic programming

### Week-by-Week Breakdown

#### **Week 9-10**: Phase 3 - Trees & Heaps (62 hours)
- **Phase 3.1-3.2**: Binary trees, BSTs (traversals, construction, LCA)
- **Phase 3.4**: Heaps & priority queues (top-K, merge K sorted lists, median finder)
- **Phase 3.5**: Tries (prefix tree, auto-complete, word search)
- **Problems**: 5 Easy + 30 Medium + 10 Hard
- **Checkpoint**: Implement BST and min-heap from scratch, solve Word Search II with tries

#### **Week 11-12**: Phases 4-5 - Graphs & DP (62 hours)
- **Phase 4.1-4.3**: Graph traversals (BFS/DFS), topological sort
- **Phase 4.4**: Shortest paths (Dijkstra, Bellman-Ford), Union-Find
- **Phase 5.1-5.3**: DP fundamentals, 1D DP, 2D DP (knapsack, LCS, edit distance)
- **Problems**: 40 Medium + 15 Hard
- **Checkpoint**: Implement Dijkstra and Union-Find, solve 5 DP problems with state definitions

**Month 3 Targets**:
- ✅ 100+ Medium/Hard problems solved
- ✅ Master tree traversals and heap operations
- ✅ Confident in graph algorithms (BFS, DFS, Dijkstra, Union-Find)
- ✅ Understand DP thinking (subproblems, memoization, tabulation)

## 🛠️ Daily Practice Structure

### Weekdays (3 hours)

**Morning (1 hour)**: 6:00-7:00 AM
- Theory and concepts (read roadmaps, watch videos)
- Learn new patterns or algorithms
- Light problem-solving (1-2 Easy problems to warm up)

**Night (2 hours)**: 8:00-10:00 PM
- Intensive coding practice (2-4 Medium problems)
- Implement data structures/algorithms from scratch
- Work on projects or system design
- Review mistakes and write notes

### Weekends (8 hours per day)

**Saturday**: Deep Dive Day
- Block 1 (2h): New topic deep dive (implement from scratch)
- Block 2 (2h): Problem-solving marathon (5-7 problems)
- Block 3 (2h): System design practice or project work
- Block 4 (2h): Review week's progress, fill knowledge gaps

**Sunday**: Practice + Planning Day
- Block 1 (2h): Mock interview or timed problem sets
- Block 2 (2h): Hard problems or advanced topics
- Block 3 (2h): Portfolio/project work
- Block 4 (1h): Weekly review + plan next week
