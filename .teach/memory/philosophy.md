## Purpose

This document defines the foundational principles, strategies, and guidelines that govern how I approach learning computer science, artificial intelligence, and software engineering. It serves as both a compass for decision-making and a living document that evolves with my growth.

**Mission**: Master comprehensive CS fundamentals, system design, AI/ML, and interview preparation within 6 months through intensive, focused, and efficient learning.

---

## Core Learning Principles

### 1. Deep Understanding Over Superficial Knowledge

**Description**: Prioritize genuine comprehension of "why" something works before memorizing "how" it works. Every concept must be understood at a fundamental level—not just memorized or copy-pasted. In a 6-month timeline, depth beats breadth.

**Rules**:
- Never move forward until you can explain a concept in your own words
- Always ask "Why does this work?" and "What problem does this solve?"
- Implement algorithms from scratch before using libraries
- Trace through examples step-by-step to build intuition
- Connect new concepts to previously learned fundamentals

---

### 2. Multimodal Learning Integration

**Description**: Leverage multiple learning modalities—visual diagrams, hands-on coding, reading documentation, and verbal explanations—to build robust mental models from different angles. Use the most efficient mode for each concept.

**Rules**:
- Start with visual diagrams or analogies to build intuition quickly
- Follow with hands-on implementation to cement understanding
- Read formal documentation to fill knowledge gaps
- Explain concepts aloud (or in writing) to test comprehension
- Use ASCII diagrams, tables, and real-world analogies liberally

---

### 3. Practice-Driven Mastery

**Description**: Knowledge without application is incomplete. Every concept must be immediately applied through coding exercises, problem-solving, or real-world projects. Code daily, without exception.

**Rules**:
- Code examples for every data structure and algorithm learned
- Solve at least 3 problems for each new pattern or technique
- Build mini-projects to apply theoretical knowledge
- Debug and fix errors hands-on rather than just reading solutions
- Maintain a personal implementation library of fundamental algorithms

---

### 4. Mistake-Driven Learning & Growth Mindset

**Description**: Mistakes are not failures—they are the most valuable learning opportunities. Embrace errors, debug thoughtfully, and extract lessons from every misstep. In an intensive timeline, fast failure leads to fast learning.

**Rules**:
- When code breaks, investigate why before asking for help
- Keep a "lessons learned" log for recurring mistakes
- Celebrate fixing bugs as learning victories
- View challenges as opportunities to grow, not obstacles
- Reflect on errors: "What did this mistake teach me?"

---

### 5. Intensive Yet Sustainable Progress

**Description**: Maintain high-intensity daily practice (3 hours weekdays, 8 hours weekends) while preserving mental clarity and avoiding burnout. Consistency and focus over marathon sessions.

**Rules**:
- **Weekdays**: 1 hour morning (theory, problem-solving), 2 hours night (implementation, practice)
- **Weekends**: 8 hours (deep dives, projects, mock interviews, comprehensive reviews)
- Use Pomodoro technique (50 min focus, 10 min break) to maintain intensity
- Take mandatory rest breaks to prevent cognitive overload
- Track energy levels and adjust difficulty of topics based on time of day
- One full rest day per week to consolidate learning and prevent burnout

## 6-Month Learning Goals

### Month 1-2: DSA Foundations & Problem-Solving
- Master core data structures (arrays, linked lists, stacks, queues, trees, graphs, heaps)
- Learn essential algorithm patterns (two pointers, sliding window, DFS/BFS, binary search)
- Solve 100+ problems (50 Easy, 40 Medium, 10 Hard)
- Achieve fluency in Big-O complexity analysis
- Complete 2 mini-projects implementing data structures from scratch

### Month 3-4: Advanced Algorithms & System Design
- Master advanced algorithms (dynamic programming, graph algorithms, greedy, backtracking)
- Study system design fundamentals (scalability, load balancing, caching, databases)
- Design 10+ systems (URL shortener, Twitter, Netflix, etc.)
- Solve 100+ Medium/Hard problems focusing on optimization
- Build 1 full-stack project demonstrating system design concepts

### Month 5-6: AI/ML Foundations & Interview Mastery
- Learn ML fundamentals (supervised learning, neural networks, optimization)
- Understand NLP and computer vision basics
- Study database design, SQL optimization, distributed systems
- Complete 10+ mock interviews (coding, system design, behavioral)
- Final sprint: 150+ total problems solved, 20+ system designs completed
- Build AI/ML portfolio project demonstrating practical application

## Time Commitment & Schedule

### Weekday Schedule (Monday - Friday)
- **Morning Session (1 hour)**: 6:00 AM - 7:00 AM or flexible morning slot
  - Theory, concepts, algorithm study
  - Reading documentation and learning new patterns
  - Light problem-solving to wake up the brain
  
- **Night Session (2 hours)**: 8:00 PM - 10:00 PM or flexible evening slot
  - Intensive coding practice
  - Problem-solving (LeetCode, system design)
  - Implementation and debugging
  - Code reviews and reflection

**Total Weekday**: 3 hours/day × 5 days = **15 hours/week**

### Weekend Schedule (Saturday - Sunday)
- **8 hours per day** with structured breaks
  - Block 1 (2 hours): Deep dive into new topic
  - Block 2 (2 hours): Intensive problem-solving
  - Block 3 (2 hours): Project work or system design
  - Block 4 (2 hours): Mock interviews, review, weekly planning

**Total Weekend**: 8 hours/day × 2 days = **16 hours/week**

### Weekly Total: ~31 hours
### Monthly Total: ~124 hours
### 6-Month Total: ~744 hours of focused learning

### Weekly Structure
- **Sunday Evening**: Weekly review (1 hour) + plan next week's focus areas
- **One Rest Day**: Optional lighter day or complete rest to prevent burnout

## Governance & Evolution

### How This Philosophy Guides Learning

1. **Prioritization**: Focus on high-impact topics aligned with interview preparation and career goals
2. **Pace Setting**: Moderate but consistent pace with weekend deep-dives for complex topics
